English 12-Keys Keyboard

The style of this keyboard reminiscent to the old 12-keys numpad keyboard on "dumb phones" (eg. old phones like Nokia). But because the function has evolved from primarily to dial number to become a keyboard, thus the design has been updated, but still based on the standard E.161 (https://en.wikipedia.org/wiki/E.161) ITU Telecommunication Standardization Sector's Recommendation that defines the arrangement of digits, letters, and symbols on telephone keypads.

For those of you who are not old enough to remember this keyboard layout, lucky you! Here's some primer:

  1. Tap button multiple time to get the letter you wanted. For example, tap the "abc" button twice to get the letter "b", and thrice to get "c".
  2. The same with the symbols and punctuations (,?!) (':") and (/()). These may be called the button rotation feature.
  3. Unlike the original numpad keyboard, which has a button delay (for example, press "abc" once to get an "a", then wait a second, press "abc" again, and you'll get "aa") there is no button delay on Keyman (if you did the previous routine, you'll end up with "b" no matter how long the wait, or even if you clicked somewhere else first). So I had to use "zero-width joiner" (on the empty button up left / number 1 location). Basically it's an invisible space, and it tells the keyboard to start the button rotation sequence from the beginning.
  4. Longpress: if you want to type a number, you can either longpress the corresponding button -- for example "abc" is "2" -- or press the "123" button to get the numeric keypad.
  5. Some other buttons also have the longpress keys, such as the "period" ("-", "@", and others), and the "spacebar" (0, qwerty keyboard, and hide keyboard button).
  6. Multi-layers: you can still access the 'qwerty' layout, in case you need it, through longpressing the space key.
  7. All Keyman Keyboards are multi-platform: PC, Android, Mac, iOS, web, etc. If you have access to the .kmn file, you can use it anywhere. But this layout is mainly for the touch experience. The physical (PC/Mac) layout are the same.
  8. You can participate by giving feedback and suggestion to the developer. Telegram/Github/Wikipedia/Twitter: @bennylin
